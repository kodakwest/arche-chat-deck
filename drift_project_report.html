Project Overview

The AgentFlow AI Chat Deck implements a local-first, multi-provider, dual-model debate interface. Built as a single HTML payload, it leverages a local Node.js proxy to route requests securely.

Unlike typical Chat UIs designed for single-actor chat completions, the Drift suite establishes an autonomous feedback loop, allowing Model A to directly challenge Model B, feeding its outputs sequentially into the opposing model's context window.

⚡ Architecture Topology:
• Client: file:// -> Browser runtime
• Core Logic: Inline vanilla ES6 modules
• Local Gateway: CORS Relay Server (Port 8787)
• Persistence: LocalStorage (Auto-save/load)

UI/UX Deficiencies & Gaps

  • 1. Delayed Feedback Loop (No streaming UI)

    The interface locks or sits static while waiting for HTTP requests to resolve. True token streaming (SSE) is necessary to keep adversarial debates visually fluid and responsive.

  • 2. Missing Theme Implementations

    Although AGENTS.md defines complete Light Mode and High Contrast tokens, the project's CSS does not contain the media queries to implement them, failing accessibility compliance.

  • 3. Sequential Thread Overload

    The vertical message list becomes difficult to read as length increases. Operators cannot view or analyze Model A and Model B's arguments side-by-side.

  • 4. Lack of Syntax Highlighting

    Model code responses are displayed in plain monochrome. This reduces readability for technical debates containing script snippets.
